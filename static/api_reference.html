API Reference

TriggersAPI is a unified RESTful interface for event ingestion, persistence, and push delivery. All endpoints accept and return application/json. Authentication is not required in development — add an Authorization: Bearer <token> header for production deployments.

Base URL http://localhost:8000
Ingestion
POST /api/v1/events Ingest Event expand_more

Accepts a JSON payload with event details, stores it with server-side metadata, and returns a structured acknowledgment. Background delivery to matching subscriptions is triggered automatically.

Request Body

Field Type Constraint Description
event_type string required Type of event, e.g. order.created
source string required Source system identifier, e.g. shopify
payload object required Arbitrary JSON object containing the event data
timestamp string optional ISO 8601 datetime — defaults to server receive time

Response 201 Created

// 201 Created
{
  "status":      "accepted",
  "event_id":    "evt_01jx3k2m...",
  "received_at": "2026-03-23T12:00:00.000Z",
  "message":     "Event ingested successfully"
}
Persistence
GET /api/v1/inbox Fetch Inbox expand_more

Returns undelivered events (status: ingested or queued) that have not been acknowledged. Supports cursor-based pagination using the received_at timestamp of the last seen event. Filter by event_type or source to narrow results.

Query Parameters

Parameter Type Default Description
event_type string — Filter results by event type
source string — Filter results by source system
limit integer 50 Max events to return (1–200)
cursor string — Pagination cursor — ISO 8601 received_at of last event

Response 200 OK

{
  "events": [ /* array of Event objects */ ],
  "count": 12,
  "next_cursor": "2026-03-23T11:59:01.123Z"
}
Acknowledgment
POST /api/v1/inbox/ack Acknowledge Events expand_more

Marks one or more events as delivered. Acknowledged events are removed from the inbox and will not appear in subsequent GET /inbox calls.

Request Body

FieldTypeConstraintDescription
event_ids string[] required Array of event IDs to acknowledge

Response 200 OK

{
  "acknowledged": ["evt_01jx...", "evt_02ab..."],
  "count": 2,
  "message": "Successfully acknowledged 2 event(s)"
}
Subscriptions
POST /api/v1/subscriptions Create Subscription expand_more

Creates a new push subscription. Matching events are delivered to endpoint_url via HTTP POST with exponential backoff retry logic.

Request Body

FieldTypeConstraintDescription
name string required Human-readable name for the subscription
endpoint_url string required HTTPS URL to deliver matched events to
event_types string[] optional Filter by event types — null matches all
sources string[] optional Filter by source systems — null matches all
retry_enabled boolean default: true Enable exponential backoff retries on failure
max_retries integer default: 5 Maximum retry attempts before marking failed
GET /api/v1/subscriptions List Subscriptions expand_more

Returns all subscriptions, optionally filtered by status (active | paused | disabled).

Query Parameters

ParameterTypeDefaultDescription
status string — Filter by subscription status
Observability
GET /api/v1/stats System Statistics expand_more

Returns aggregated system-wide statistics including event counts by status, active subscriptions, and delivery success rate.

Response 200 OK

  "events_by_status": {

    "delivered": 990,
    "failed":    24

  "total_subscriptions": 5,
  "active_subscriptions": 4,
  "success_rate":        97.65,
  "avg_latency_ms":     43.2
}
GET /health Health Check expand_more

Lightweight liveness probe suitable for load balancers and orchestration platforms.

Response 200 OK

{
  "status":  "healthy",
  "service": "triggers-api",
  "version": "1.0.0"
}
