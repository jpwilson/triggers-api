TriggersAPI — Webhook Event Bus

A lightweight, production-ready event ingestion and delivery platform built with FastAPI and SQLite.

Overview

TriggersAPI is a webhook event bus that decouples event producers from event consumers. It provides a structured pipeline for ingesting, queuing, and reliably delivering events to registered HTTP subscribers.

Event Lifecycle

input 1. Ingest

Events arrive via POST /api/v1/events/{trigger_id}. Each event is assigned a UUID, timestamped, and persisted to SQLite in WAL mode.

sync 2. Queue

The background delivery worker polls for queued events and fan-outs to all active subscriptions. Pending deliveries are tracked with status.

send 3. Deliver

Events are POSTed to subscriber webhook URLs. Failed deliveries are retried with exponential backoff (1s, 2s, 4s, 8s, 16s). At-least-once guarantee.

Subscription Model

Subscribers register a webhook URL and optionally filter by trigger ID or event type. Each subscription can be independently paused, resumed, or deleted. The system supports fan-out to multiple subscribers per trigger, enabling pub/sub patterns.

AI Integration

TriggersAPI includes an AI chat layer powered by OpenRouter. The POST /api/v1/chat endpoint routes to the configured LLM model with optional Langfuse observability tracing. Conversation context is maintained per session.
